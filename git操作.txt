如何将本地的仓库上传到github仓库中
首先先新建一个空的文件夹
在空的文件夹中打开git bash 右键git bash 
然后再敲入git init  然后git add 文件 这步操作是将这个文件放入到仓库的暂存区 如果添加的文件和修改的文件很多可以直接git add . 将所用的文件放入到暂存区中
git commit -m '所有的计算机基础的ppt'  
git log查看提交的文件的日志  有一个commit id 03eee2bb790c05cfe492bfd3e199cbb32e878e87 
git log --stat 查看每次提交了哪些修改文件  
配置验证登录ssh-keygen -t rsa -C 1931665318@qq.com 一直按enter 然后到/c/Users/asus/.ssh/id_rsa.pub找到公钥回到github中点击头像然后setting 找到ssh new sshkey 随便取一个名字然后把id_rsa.pub中的内容复制上去
然后找到你要上传的仓库git@github.com:Bais-art/teaching.git    然后 git remote add teaching git@github.com:Bais-art/teaching.git 一旦配置了ssh 以后别的上传别的仓库也不需要配置ssh了。
最后git push teaching master 作为主干上传上去

怎么拉取代码  首先在本地建一个空的文件夹然后 打开git bash 输入git init 然后git pull git@github.com:Bais-art/teaching.git master 把它的主干拉取下来 然后将这个仓库里的git操作.txt文件修改再上传上teaching仓库中。 1：git add .  2：git commit -m '修改git操作.txt'  3：git push git@github.com:Bais-art/teaching.git master  如果发生了改变还需要先拉取这个仓库然后再把本地仓库新增的文件上传上去否则就会拒绝上传。

git log可以看每次提交的信息 
git log --stat可以看每次提交的时候都提交了哪些文件
git diff 某次提交的id就可以查看修改文件的内容
git reset --hard 某次提交的id 就可以把代码回退到指定的节点
git branch 就可以看到有什么分支 分支的概念比如同一个项目一个是英文的app和一个中文的app每次更新的时间不同，
master分支用于保存经过测试的稳定代码，用来在发布新版本的时候使用。一般情况下是不能修改的，如果要开发出新的功能需要在master基础上复制出来develop 直接
git check -b develop 创建新的分支 等开发和测试完毕后要发布的时候把develop分支合并到master上
 首先1：git add 2：git commit 3：git checkout  master 在master上git merge develop就完成了合并
